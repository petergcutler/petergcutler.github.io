From 2015-2017, I worked for Crowdskout as a founding software engineer, where we made CRM tools for political and advocacy campaigns. I specialized in developing its systems for audience segmenting, analytics, and data visualization. Crowdskout was acquired by Murmuration in 2018.

Core Features for Crowdskout

The engineering work I did for Crowdskout often didn't involve a visual component. One of my main responsibilities was developing the state machine for our app, to maintain data configuration as users moved among complex, multi-step flows. But my work over time trended towards data analytics, and inolved more well-designed front-end components.

User Profiles and Data Fields

One core feature I worked on was Crowdskout's user profile and included fields. As an advocacy CRM, Crowdskout provided hundreds of possible data fields on our user profiles that could be used to evaluate and sort voting populations.

User Segmenting

Fueled by these detailed user profiles, I also built custom tooling to filter, parse, and save collections of profiles that match queries into segments. Segments could be used across Crowdskout to track and visualize campaign engagements.

Interactive Forms for Canvassing and Polling

One way that Crowdskout users can interact with their profiles is to deliver surveys and forms for a variety of circumstances. I overhauled the form system and added a system for conditionally branching questions to facilitate rich survey creation entirely within the Crowdskout platform.

A Crowdskout user may use forms to ask any number of questions. These questions in turn can fuel the creating or updating of any of the hundreds of Crowdskout profile fields that exist in their database.

Building Data Visualizations with D3 and amCharts

One of my roles was to research and build the foundations of Crowdskout's data visualization system. I designed it so that a novice user could quickly learn and operate it and deliver complex visualizations without dedicated training. I built the system using a mix of D3 and a library called amCharts, also built on D3.

Using a multi-step editor, a user could select an audience segment, criteria, and a graph type, and create a graph that could be customized or exported in a variety of ways.
